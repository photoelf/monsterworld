# Карманная Братва (monsterworld)

Покемоноподобная игра «на коленке»: ванильный JS без сборки и зависимостей. Владелец: photoelf (TG id 84682640).

## Деплой и адреса

- **Игра**: https://photoelf.github.io/monsterworld/ — GitHub Pages, деплой = `git push origin main`, докатывается ~30–60 с. Проверка: `curl sw.js | grep monsterworld-vNN`.
- **Mini App**: бот **@poketmons_bot** (название «Карманная Братва»). Токен бота — в Cloudflare Secrets (BOT_TOKEN); он засветился в чате 17.07.2026 — при ротации: `wrangler secret put BOT_TOKEN` + повторный `setWebhook` с WEBHOOK_SECRET.
- **API**: Cloudflare Worker `monsterworld-api` → https://monsterworld-api.photoelf.workers.dev (= API_BASE в js/net.js). Код в `backend/worker.js`. Деплой: `cd backend && env -u CLOUDFLARE_API_TOKEN npx wrangler deploy` (**важно**: в окружении лежит битый CLOUDFLARE_API_TOKEN — всегда `env -u`; auth = OAuth wrangler login). KV namespace: `6aa2b5ec131542dbbd2a5c6a388d63e0` (binding SNAPS).
- **ОБЯЗАТЕЛЬНО**: при любом изменении фронта поднимать версию `CACHE` в `sw.js` (номер смотреть в самом sw.js), иначе PWA-клиенты остаются на старом коде. Пользователю говорить «перезапусти мини-апп дважды».
- **SW-обновления атомарные** (после аварии v19): install качает ассеты с `cache:'reload'` (GH Pages кэширует по 10 мин — иначе в кэш SW попадает микс старых/новых файлов и main() падает), fetch — строго cache-first БЕЗ фоновой дозаписи. Не возвращать stale-while-revalidate.

## Файлы

- `index.html` — вся разметка + весь CSS инлайн. `js/`: `tg.js` (Telegram: фулскрин, отступы, CloudStorage-сейвы, рега ника), `net.js` (API: снапшоты команд, Stars, промокоды), `util.js` (в т.ч. звук `sfx`/`SOUND_ON` и `keyHint`), `data.js` (виды/спрайты/умения), `world.js` (генерация мира, города), `battle.js`, `main.js` (всё остальное), `pvp.js`.
- `backend/worker.js` + `wrangler.toml`.

## Ключевые механики и решения

- **Терминология**: существа братишка/брат/братан по стадиям (`stageWord()`), панели «Братва» (команда), «Карман» (хранилище), «Братопедия». Супертренеры Иван/Полина/Андрей/Миша = 50% лидеров городов.
- **Города**: Вороной-ячейки 96×96 (`world.js`), компактный круг r=16–21, ровно по 1 арене/лавке/фонтану/башне/питомнику + 1–2 доски, дороги между живыми центрами, процедурные имена. Центр на воде → города нет (`dead`).
- **Мобильный режим**: `body.mobile`; `IS_MOBILE` — `let`, приоритет: URL `?desktop`/`?mobile` → настройка `localStorage('mw-mode')` (`FORCED_MODE`) → автодетект (touch / tgWebApp-hash); `initTelegram()` дожимает в true внутри Telegram, но уважает форс десктопа. Фулскрин-канвас с динамическим VIEW (`resizeCanvas`), джойстик 360° (`joy`), тач-кнопки видны только при `data-state="world"`. Отступы: CSS-переменные `--safe-*`. Десктоп (обычный браузер): `applyDesktopZoom` растягивает 960px до 3×.
- **Настройки** (`#settings-panel`, `toggleSettings`/`renderSettings`): вход — Esc из мира на десктопе (клавиша `,` тоже), кнопка ⚙️ в тач-меню на мобиле. Внутри: звук (`SOUND_ON`, `localStorage('mw-sound')`), режим Авто/Мобильный/Десктоп (пишет `mw-mode` и перезагружает; вычищает `?desktop`/`?mobile` из URL), «На весь экран» (Fullscreen API; скрыта на мобиле и в TMA — там не работает). На мобиле `stripKeyHints()` срезает «(Esc)»/«(Tab)» со статичных кнопок, в динамических текстах — `keyHint('B')`.
- **Карта**: кнопки телепорта к фонтанам подписываются именем города (`World.cityInfoAt`) + расстояние в км (тайлы ÷ 10).
- **Бой**: покемон-раскладка (враг ↗, свой ↙ со спины — `speciesSprite(...,back)` не рисует глазки). Анимации CSS: `idlebob` (дыхание), `lunge-r/l` (выпад), `shake` — **только классовая специфичность, id-селекторы сломают** (уже наступали). Отменяемые меню имеют кнопку «‹ Назад» (иначе софтлок на таче).
- **Кастомные спрайты** (`customSprite` = base64 PNG ≤24×24, смотрит ВЛЕВО): рендер масштабируется к размеру стадии через `monSpriteDims` (12/15/18px); в бою свой флипается вправо, спутник флипается по позиции игрока. `Image` из data-URL грузится асинхронно — `monMiniCanvas` дорисовывает канвас по `load` (иначе видно старый облик до перезахода). Едет в сейв/облако/обмены с другом; в публичный пул НЕ попадает (whitelist воркера).
- **Облачные сейвы**: Telegram CloudStorage, чанки `sv_0..N` + `sv_meta` (≤3500 симв/ключ), дебаунс 20 с, свежий по `ts` побеждает при запуске. Ник: KV-ключ `profile`, дефолт из TG username.
- **Живые соперники**: после боёв команда уходит POST /team (аноним `clientId` из localStorage), 40% тренерских боёв берут чужой снапшот («Тренер Братвы <ник>»), уровни клампятся к местности, статы пересчитывает `tradeMonRevive` (анти-чит). Свой снапшот фильтруется по нику (clientId на каждом устройстве свой, сейв общий — иначе дерёшься сам с собой).
- **Побег**: «Бежать» работает и против тренеров/лидеров/башни (та же формула от скорости); тренер не помечается побеждённым — можно переиграть.
- **Палитры заказных** (`m.palette` = id из `MON_PALETTES` в data.js): перекрашивает процедурный спрайт (`speciesSprite(..., palId)`), едет в сейв/обмены/пул (воркер валидирует по списку); shiny сильнее палитры.

## Монетизация (работает)

- **Товары** — таблица `PRODUCTS` в worker.js (`spr`/`wrd`/`mon`), цены синхронно в net.js. Флоу: `/invoice {product}` → `TG.openInvoice` → вебхук `/bot` (payload `<prod>:<clientId>`) → клиент поллит статус.
  - `spr` **10 Stars** — свои PNG-спрайты (KV `unlock:<id>`, поллинг `/unlock`).
  - `wrd` **15 Stars** — гардероб: перекраски футболки/волос/кожи (7 оттенков: белый, 5 телесных, чёрный) + аксессуары кепка/очки/корона (`OUTFIT_SHIRTS/HAIRS/SKINS/ACCS`, `G.outfit` в сейве, панель из настроек и лавки; KV `wrd:<id>`, `/unlock {product:'wrd'}`).
  - `mon` **25 Stars за штуку** — генератор заказных братишек: тип/окрас/имя, предпросмотр цепочки, `findSpeciesForOrder` (3 стадии, сумма базы 300–350 — не выше легендарок), **всегда шайни** (+18% статов; выбранная палитра в отрисовке сильнее золота шайни). Кредиты в KV `mon:<id>` (счётчик), `/mongen` (статус) + `/mongen/claim` (списание); VIP — без ограничений.
- **Лавка** — вкладки `SHOP_CATS` (зелья/снаряжение/амулеты/⭐ Особое); товары с `cat` в `SHOP_ITEMS`; в «Особом» витрины всех Stars-товаров.
- **VIP** (бесплатно): KV-ключи `vip:<tg-id>`; клиент шлёт подписанный `initData`, воркер проверяет HMAC (bot token). В списке: 84682640 (владелец), 130698362, 499446622. Бот-команда `/id` говорит юзеру его id.
- **Промокоды**: KV `promo:<CODE>` = `{"uses":N}`, эндпоинт `/redeem`, кнопка «🎟 Промокод» в витрине (работает и в браузере без TG). Действующий: **BRATVA-OG** (30 активаций).

## Планы (согласованы с владельцем)

1. **Быстрые сейв/лоад для сейвскамеров** — кандидат в премиум за Stars: сейв перед каждой атакой в бою + загрузка, если результат не понравился.
2. **Маунт на братане** (3-я стадия эволюции) — скорость 1.5× от бега.
3. **Система хинтов**: инфо-сообщения (типа «нужен водяной братишка 15 ур., чтобы плавать») показывать не каждый раз, а дозированно.
4. **Премиум-маунт «Электросамокат»** — возможно за Stars.
5. Идеи на потом: показывать кастомные спрайты другим игрокам (нужна модерация); полный кастомный PNG игрока (нужны 4 направления).

Сделано из прежних планов: гардероб игрока (wrd) и генератор заказных братишек (mon) — см. «Монетизация».
