# Карманная Братва (monsterworld)

Покемоноподобная игра «на коленке»: ванильный JS без сборки и зависимостей. Владелец: photoelf (TG id 84682640).

## Деплой и адреса

- **Игра**: https://photoelf.github.io/monsterworld/ — GitHub Pages, деплой = `git push origin main`, докатывается ~30–60 с. Проверка: `curl sw.js | grep monsterworld-vNN`.
- **Mini App**: бот **@poketmons_bot** (название «Карманная Братва»). Токен бота — в Cloudflare Secrets (BOT_TOKEN). Ротирован 18.07.2026 (прежний светился в чате 17–18.07, мёртв). Заливать секрет надёжнее через Cloudflare Dashboard (Workers → monsterworld-api → Settings → Variables and Secrets): `wrangler secret put` в этом харнессе не получает интерактивный ввод и заливает пустоту. Проверка живости токена: `curl -X POST .../invoice -d '{"id":"testclient123","product":"spr"}'` → должна прийти `link`, а не `tg error`. Revoke токена НЕ сбрасывает вебхук (URL привязан к боту, WEBHOOK_SECRET тот же) — `setWebhook` переставлять нужно только при смене URL/секрета.
- **API**: Cloudflare Worker `monsterworld-api` → https://monsterworld-api.photoelf.workers.dev (= API_BASE в js/net.js). Код в `backend/worker.js`. Деплой: `cd backend && env -u CLOUDFLARE_API_TOKEN npx wrangler deploy` (**важно**: в окружении лежит битый CLOUDFLARE_API_TOKEN — всегда `env -u`; auth = OAuth wrangler login). KV namespace: `6aa2b5ec131542dbbd2a5c6a388d63e0` (binding SNAPS).
- **ОБЯЗАТЕЛЬНО**: при любом изменении фронта поднимать версию `CACHE` в `sw.js` (номер смотреть в самом sw.js), иначе PWA-клиенты остаются на старом коде. Пользователю говорить «перезапусти мини-апп дважды».
- **SW-обновления атомарные** (после аварии v19): install качает ассеты с `cache:'reload'` (GH Pages кэширует по 10 мин — иначе в кэш SW попадает микс старых/новых файлов и main() падает), fetch — строго cache-first БЕЗ фоновой дозаписи. Не возвращать stale-while-revalidate. Клиент слушает `controllerchange` и делает один `location.reload()` при активации нового SW (флаг `hadController` гасит лишний reload на первой установке) — свежий код подтягивается за ОДИН заход, «перезапусти дважды» больше не нужно. Диагностика застрявшего клиента: `curl sw.js | grep vNN` (прод-версия) + проверить, что все файлы `ASSETS` отдают 200 (один 404 валит install нового SW — обновление не наступит никогда).

## Файлы

- `index.html` — вся разметка + весь CSS инлайн. `js/`: `tg.js` (Telegram: фулскрин, отступы, CloudStorage-сейвы, рега ника), `net.js` (API: снапшоты команд, Stars, промокоды), `util.js` (в т.ч. звук `sfx`/`SOUND_ON` и `keyHint`), `data.js` (виды/спрайты/умения), `world.js` (генерация мира, города), `battle.js`, `main.js` (всё остальное), `pvp.js`.
- `backend/worker.js` + `wrangler.toml`.

## Ключевые механики и решения

- **Терминология**: существа братишка/брат/братан по стадиям (`stageWord()`), панели «Братва» (команда), «Карман» (хранилище), «Братопедия». Супертренеры Иван/Полина/Андрей/Миша = 50% лидеров городов.
- **Братопедия**: `renderDexGrid()` — сетка видов; тап по пойманному → `openDexCard(seed)` (крупный спрайт, цепочка эволюций, задатки, описание). Описание — `speciesFlavor(seed)` в data.js: собирается ПРОЦЕДУРНО из черт вида (тип/доминантный стат/число стадий) детерминированно по сиду, БЕЗ AI (ноль стоимости, у вида всегда один текст). Тон — тёплый уличный. Расширять — добавлять фрагменты в `FLAVOR_TYPE/STAT/STAGE`.
- **Города**: Вороной-ячейки 96×96 (`world.js`), компактный круг r=16–21, ровно по 1 арене/лавке/фонтану/башне/питомнику + 1–2 доски, дороги между живыми центрами, процедурные имена. Центр на воде → города нет (`dead`).
- **Мобильный режим**: `body.mobile`; `IS_MOBILE` — `let`, приоритет: URL `?desktop`/`?mobile` → настройка `localStorage('mw-mode')` (`FORCED_MODE`) → автодетект (touch / tgWebApp-hash); `initTelegram()` дожимает в true внутри Telegram, но уважает форс десктопа. Фулскрин-канвас с динамическим VIEW (`resizeCanvas`), джойстик 360° (`joy`), тач-кнопки видны только при `data-state="world"`. Отступы: CSS-переменные `--safe-*`. Десктоп (обычный браузер): `applyDesktopZoom` растягивает 960px до 3×.
- **Настройки** (`#settings-panel`, `toggleSettings`/`renderSettings`): вход — Esc из мира на десктопе (клавиша `,` тоже), кнопка ⚙️ в тач-меню на мобиле. Внутри: звук (`SOUND_ON`, `localStorage('mw-sound')`), раскладка джойстика (`JOY_SIDE`/`mw-joyside`, дефолт `right` — стик справа/кнопки слева; класс `body.joy-right` зеркалит CSS; кнопка только на мобиле), режим Авто/Мобильный/Десктоп (пишет `mw-mode` и перезагружает; вычищает `?desktop`/`?mobile` из URL), «На весь экран» (Fullscreen API; скрыта на мобиле и в TMA — там не работает). На мобиле `stripKeyHints()` срезает «(Esc)»/«(Tab)» со статичных кнопок, в динамических текстах — `keyHint('B')`.
- **Карта**: кнопки телепорта к фонтанам подписываются именем города (`World.cityInfoAt`) + расстояние в км (тайлы ÷ 10).
- **Дозированные хинты** (`hint(id, text, maxShow=3)` в main.js): обучающий toast показывается не более maxShow раз за игру, потом молчит. Счётчик `G.hints` в сейве. Применён к `surf` (нужен водный 15+) и `rod` (нужна удочка).
- **Бой**: покемон-раскладка (враг ↗, свой ↙ со спины — `speciesSprite(...,back)` не рисует глазки). Анимации CSS: `idlebob` (дыхание), `lunge-r/l` (выпад), `shake` — **только классовая специфичность, id-селекторы сломают** (уже наступали). Отменяемые меню имеют кнопку «‹ Назад» (иначе софтлок на таче).
- **Кастомные спрайты** (`customSprite` = base64 PNG ≤24×24, смотрит ВЛЕВО): рендер масштабируется к размеру стадии через `monSpriteDims` (12/15/18px); в бою свой флипается вправо, спутник флипается по позиции игрока. `Image` из data-URL грузится асинхронно — `monMiniCanvas` дорисовывает канвас по `load` (иначе видно старый облик до перезахода). Едет в сейв/облако/обмены с другом; в публичный пул НЕ попадает (whitelist воркера).
- **Облачные сейвы**: Telegram CloudStorage, чанки `sv_0..N` + `sv_meta` (≤3500 симв/ключ), дебаунс 20 с, свежий по `ts` побеждает при запуске. Ник: KV-ключ `profile`, дефолт из TG username.
- **Живые соперники**: после боёв команда уходит POST /team (аноним `clientId` из localStorage), 40% тренерских боёв берут чужой снапшот («Тренер Братвы <ник>»), уровни клампятся к местности, статы пересчитывает `tradeMonRevive` (анти-чит). Свой снапшот фильтруется по нику (clientId на каждом устройстве свой, сейв общий — иначе дерёшься сам с собой).
- **Безопасность недоверенного текста**: клички/имена умений из чужих кодов обмена/PvP и импортированных кодов сейва рендерятся через `innerHTML` (много мест). Коды идут МИМО API (checksum+соль `TRADE_SALT` лежат в клиенте — валидный код собирает кто угодно), поэтому клиент — единственный барьер: `safeNick`/`safeMoveName` в `reviveOwnedMon` и `tradeMonRevive` режут `<>&"'` (зеркалят фильтры воркера). Не отключать. Свой ник (`playerNick`) рендерится через `textContent` — не трогать. `tradeDecode`/`importSaveCode` отбрасывают коды сверх лимита длины (300КБ/4МБ) до дорогого декода — защита от вставки-DoS.
- **Побег**: «Бежать» работает и против тренеров/лидеров/башни (та же формула от скорости); тренер не помечается побеждённым — можно переиграть.
- **Сейвскам** — премиум-товар `scum` **10⭐** (`scumUnlocked`, KV `scum:<id>`; функции гейтятся `SCUM_ON && scumUnlocked`). Тумблер `SCUM_ON`/`mw-scum` в настройках. Две части:
  - Работает ТОЛЬКО в бою (эмуляторный сейв-стейт, `Battle.battleSave`/`battleReload`, кнопки `#bt-scum` вверху боевого экрана): снапшот ОЗ/ПП/статусов обеих команд + сфер + активных бойцов в память. Откат мутирует существующие объекты (ссылки те же) и реджектит текущий `await` маркером `BATTLE_RELOAD` — цикл боя ловит в `try/catch` и перезапускает ход (`say`/`menu` резолвят/реджектят). Переигрывай любой удар. Вне-боевого сейв/отката НЕТ (убрали намеренно — не нужен).
- PvP-реплей (`pvpShowReplay`) — кнопка «⏭ Скип» (`finish()` вываливает лог и показывает итог сразу).
- **KV-экономия**: разовые покупки при старте — одним запросом `POST /status` (`netCheckStatus` → spr/scoot/scum/vip) вместо запроса на товар. «Живой» соперник (`netFetchRival`) тянется ЛЕНИВО при первом бою (`netTakeRival`), НЕ при старте. Не возвращать предзагрузку/отдельные проверки — жжёт KV-операции (упирались в дневной лимит).
- **Маунты** (`activeMount()` в main.js): под игроком рисуется маунт и множится скорость. `water` — плывём на водном братишке 15+ (пассивно, по факту нахождения на воде + визуал качки); `scooter` — премиум-самокат на суше ×1.8 (`scootUnlocked` + `G.scootOn`); `brother` — лидер-братан (stage 2) на суше ×1.5. **Наземный маунт — только при нажатом ускорении** (`keys.has('Shift')`/кнопка бега): пешком идёшь сам, нажал бег — едешь на доступном маунте. Кнопка бега `#t-run` меняет иконку в `updateHUD`: 🛴 самокат / 🐎 братан / 🏃 просто бег (`landMountKind()`). Приоритет вода → самокат → братан; при water/brother спутник не бежит рядом (он и есть маунт).
- **Кнопка рыбалки** `#t-fish` видна только с удочкой И когда `waterNearby()` (вода в соседнем тайле) — обновляется в `updateHUD`.
- **PvP через Telegram** (без копирования кодов, `pvp.js` + воркер): A жмёт «⚔️ Вызвать через Telegram» → `netPvpCreate` кладёт команду в KV `pvp:<id>` (TTL 3 дня) → `tgSharePvp` открывает штатный share-пикер (`t.me/share/url` со ссылкой `t.me/poketmons_bot?startapp=pvp<id>`). Открытый вызов — первый принявший. B кликает ссылку → `START_PARAM` (`initDataUnsafe.start_param`) → `handlePvpDeepLink` → `openPvpFromLink`/`pvpTgHandle` → принимает, `pvpSimulate` считает бой, `netPvpaccept` шлёт вердикт + пуш A (`sendMessage` с кнопкой «Смотреть бой», best-effort). Симуляция детерминированная (общий сид) — обе стороны видят один бой. **Награду начисляет клиент** (`pvpAward`: победителю деньги+опыт, проигравший без потерь) один раз на вызов (флаг `pvpgot:<id>` в `usedTrades`). Поля запроса: `id`=clientId (для `buyerKey`), `ch`=id вызова — НЕ путать. `mySide` (A/B) сервер вычисляет по `buyerKey`. Кодовый PvP (`pvpChallengeFlow`, tradeEncode) остался как запасной для не-TMA.
- **Палитры заказных** (`m.palette` = id из `MON_PALETTES` в data.js): перекрашивает процедурный спрайт (`speciesSprite(..., palId)`), едет в сейв/обмены/пул (воркер валидирует по списку). В отрисовке палитра **сильнее** золота шайни (иначе купленный окрас терялся бы — заказные всегда шайни); у шайни с палитрой акцентные пиксели белые.
- **Спрайт игрока**: `makePersonSprite(shirt, hair, acc, skin)` — лист 4 направления × 2 кадра по 16×16; аксессуары рисуются поверх волос/глаз для каждого направления. Наряд `G.outfit {shirt, hair, acc, skin}` в сейве, применяется `applyOutfit()`.

## Монетизация (работает)

- **Товары** — таблица `PRODUCTS` в worker.js (`spr`/`mon`/`scoot`; `wrd` — легаси) + поштучные аксессуары `acc`. Цены синхронно в net.js. Флоу: `/invoice {product, initData}` → `TG.openInvoice` → вебхук `/bot` (payload `<prod>:<ключ>`, у аксессуаров `acc:<item>:<ключ>`) → клиент поллит статус.
- **Ключ покупателя = tg-id** (безопасность): `buyerKey(env, body)` в worker.js — для TMA `'tg'+verifyInitData().id` (покупка привязана к аккаунту Telegram, НЕ к clientId/сейву — передача сейва чужому не разблокирует платное), для браузера фоллбэк на `clientId`. Все запросы покупок шлют `initData` (`tgInitData()` в net.js). Сейв (`buildSaveData`) не содержит флагов покупок — только тумблер `scootOn`. Локальные кэш-флаги переименованы в `mw-*-unlocked2`/`mw-accs2` (миграция: сбрасывает старую авто-VIP-выдачу, клиент перепроверяет сервер).
- **VIP покупает явно**: статус-проверки (`/unlock`, `/accs`, `/mongen`) больше НЕ авто-выдают по VIP. VIP получает товар бесплатно только по нажатию «Купить» — `/invoice` возвращает `{granted:true}` и сразу пишет флаг (клиент обрабатывает `granted` как успешную покупку). Не-VIP получает `link` и платит.
- **Порядок деплоя при новых товарах/эндпоинтах: сначала воркер, потом фронт** (старый `/invoice` молча продаст не то). После `wrangler deploy` эндпоинты доезжают с лагом ~10–30 с — не паниковать при первом 404.
  - `spr` **10 Stars** — свои PNG-спрайты (KV `unlock:<id>`, поллинг `/unlock`).
  - **Гардероб**: перекраски футболки/волос/кожи (7 оттенков: белый, 5 телесных, чёрный) — **бесплатно всем**; аксессуары — **поштучно 1–5 Stars** (кепка 1 / очки 2 / корона 5; `ACC_PRICES` синхронно в net.js и worker.js). Куплено = KV `accs:<clientId>` (JSON-массив), статус POST `/accs`, счёт `/invoice {product:'acc', item}`. Старый анлок `wrd:<id>` (15⭐) и VIP дают все аксессуары (гранфазеринг). Вход в панель из настроек и лавки. **Новый аксессуар = 4 правки**: цена в `ACC_PRICES` (worker.js И net.js), имя в `ACC_NAMES` (worker.js) и `OUTFIT_ACCS` (main.js), отрисовка в `makePersonSprite` (все 4 направления!); воркер деплоить РАНЬШЕ фронта. План: креативные аксессуары с отсылками к играм, надписи на майках.
  - `mon` **25 Stars за штуку** — генератор заказных братишек: тип/окрас/имя, предпросмотр цепочки, `findSpeciesForOrder` (3 стадии, сумма базы 300–350 — не выше легендарок), **всегда шайни** (+18% статов; выбранная палитра в отрисовке сильнее золота шайни). Кредиты в KV `mon:<id>` (счётчик), `/mongen` (статус) + `/mongen/claim` (списание); VIP — без ограничений.
  - `scoot` **50 Stars** — премиум-маунт «Электросамокат» (once-unlock KV `scoot:<id>`, `/unlock {product:'scoot'}`). Флаг `scootUnlocked` в net.js, тумблер `G.scootOn` в сейве. См. `activeMount()` в механиках.
  - `scum` **10 Stars** — режим сейвскамера (once-unlock KV `scum:<id>`). Флаг `scumUnlocked`. См. «Сейвскам» в механиках.
- **Лавка** — вкладки `SHOP_CATS` (зелья/снаряжение/амулеты/⭐ Особое); товары с `cat` в `SHOP_ITEMS`; в «Особом» витрины всех Stars-товаров.
- **VIP** (бесплатно): KV-ключи `vip:<tg-id>`; клиент шлёт подписанный `initData`, воркер проверяет HMAC (bot token). В списке: 84682640 (владелец), 130698362, 499446622. Бот-команда `/id` говорит юзеру его id.
- **Промокоды**: KV `promo:<CODE>` = `{"uses":N}`, эндпоинт `/redeem`, кнопка «🎟 Промокод» в витрине (работает и в браузере без TG). Действующий: **BRATVA-OG** (30 активаций).
