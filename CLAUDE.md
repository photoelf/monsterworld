# Карманная Братва (monsterworld)

Покемоноподобная игра «на коленке»: ванильный JS без сборки и зависимостей. Владелец: photoelf (TG id 84682640).

## Деплой и адреса

- **Игра**: https://photoelf.github.io/monsterworld/ — GitHub Pages, деплой = `git push origin main`, докатывается ~30–60 с. Проверка: `curl sw.js | grep monsterworld-vNN`.
- **Mini App**: бот **@poketmons_bot** (название «Карманная Братва»). Токен бота — в Cloudflare Secrets (BOT_TOKEN). Ротирован 18.07.2026 (прежний светился в чате 17–18.07, мёртв). Заливать секрет надёжнее через Cloudflare Dashboard (Workers → monsterworld-api → Settings → Variables and Secrets): `wrangler secret put` в этом харнессе не получает интерактивный ввод и заливает пустоту. Проверка живости токена: `curl -X POST .../invoice -d '{"id":"testclient123","product":"spr"}'` → должна прийти `link`, а не `tg error`. Revoke токена НЕ сбрасывает вебхук (URL привязан к боту, WEBHOOK_SECRET тот же) — `setWebhook` переставлять нужно только при смене URL/секрета.
- **API**: Cloudflare Worker `monsterworld-api` → https://monsterworld-api.photoelf.workers.dev (= API_BASE в js/net.js). Код в `backend/worker.js`. Деплой: `cd backend && env -u CLOUDFLARE_API_TOKEN npx wrangler deploy` (**важно**: в окружении лежит битый CLOUDFLARE_API_TOKEN — всегда `env -u`; auth = OAuth wrangler login). KV namespace: `6aa2b5ec131542dbbd2a5c6a388d63e0` (binding SNAPS) — покупки/редкие ключи. **D1** `monsterworld` `e31494fb-47ab-47e9-bd12-d37edc55bb48` (binding DB) — горячий игровой поток (команды/лидерборд/PvP), схема в `backend/schema.sql`. Применить схему в новое окружение: `wrangler d1 execute monsterworld --remote --file=backend/schema.sql`.
- **ОБЯЗАТЕЛЬНО**: при любом изменении фронта поднимать версию `CACHE` в `sw.js` (номер смотреть в самом sw.js), иначе PWA-клиенты остаются на старом коде. Пользователю говорить «перезапусти мини-апп дважды».
- **SW-обновления атомарные** (после аварии v19): install качает ассеты с `cache:'reload'` (GH Pages кэширует по 10 мин — иначе в кэш SW попадает микс старых/новых файлов и main() падает), fetch — строго cache-first БЕЗ фоновой дозаписи. Не возвращать stale-while-revalidate. Клиент слушает `controllerchange` и делает один `location.reload()` при активации нового SW (флаг `hadController` гасит лишний reload на первой установке) — свежий код подтягивается за ОДИН заход, «перезапусти дважды» больше не нужно. Диагностика застрявшего клиента: `curl sw.js | grep vNN` (прод-версия) + проверить, что все файлы `ASSETS` отдают 200 (один 404 валит install нового SW — обновление не наступит никогда).

## Файлы

- `index.html` — вся разметка + весь CSS инлайн. `js/`: `tg.js` (Telegram: фулскрин, отступы, CloudStorage-сейвы, рега ника), `net.js` (API: снапшоты команд, Stars, промокоды), `util.js` (в т.ч. звук `sfx`/`SOUND_ON` и `keyHint`), `data.js` (виды/спрайты/умения), `world.js` (генерация мира, города), `battle.js`, `main.js` (всё остальное), `pvp.js`.
- `backend/worker.js` + `wrangler.toml`.

## Ключевые механики и решения

- **Терминология**: существа братишка/брат/братан по стадиям (`stageWord()`), панели «Братва» (команда), «Карман» (хранилище), «Братопедия». Супертренеры Иван/Полина/Андрей/Миша = 50% лидеров городов.
- **Братопедия**: `renderDexGrid()` — сетка видов; тап по пойманному → `openDexCard(seed)` (крупный спрайт, цепочка эволюций, задатки, описание). Описание — `speciesFlavor(seed)` в data.js: собирается ПРОЦЕДУРНО из черт вида (тип/доминантный стат/число стадий) детерминированно по сиду, БЕЗ AI (ноль стоимости, у вида всегда один текст). Тон — тёплый уличный. Расширять — добавлять фрагменты в `FLAVOR_TYPE/STAT/STAGE`.
- **Города**: Вороной-ячейки 96×96 (`world.js`), компактный круг r=16–21, ровно по 1 арене/лавке/фонтану/башне/питомнику + 1–2 доски, дороги между живыми центрами, процедурные имена. Центр на воде → города нет (`dead`).
- **Мобильный режим**: `body.mobile`; `IS_MOBILE` — `let`, приоритет: URL `?desktop`/`?mobile` → настройка `localStorage('mw-mode')` (`FORCED_MODE`) → автодетект (touch / tgWebApp-hash); `initTelegram()` дожимает в true внутри Telegram, но уважает форс десктопа. Фулскрин-канвас с динамическим VIEW (`resizeCanvas`), джойстик 360° (`joy`), тач-кнопки видны только при `data-state="world"`. Отступы: CSS-переменные `--safe-*`. Десктоп (обычный браузер): `applyDesktopZoom` растягивает 960px до 3×.
- **Настройки** (`#settings-panel`, `toggleSettings`/`renderSettings`): вход — Esc из мира на десктопе (клавиша `,` тоже), кнопка ⚙️ в тач-меню на мобиле. Внутри: звук (`SOUND_ON`, `localStorage('mw-sound')`), раскладка джойстика (`JOY_SIDE`/`mw-joyside`, дефолт `right` — стик справа/кнопки слева; класс `body.joy-right` зеркалит CSS; кнопка только на мобиле), режим Авто/Мобильный/Десктоп (пишет `mw-mode` и перезагружает; вычищает `?desktop`/`?mobile` из URL), «На весь экран» (Fullscreen API; скрыта на мобиле и в TMA — там не работает). На мобиле `stripKeyHints()` срезает «(Esc)»/«(Tab)» со статичных кнопок, в динамических текстах — `keyHint('B')`.
- **Карта**: кнопки телепорта к фонтанам подписываются именем города (`World.cityInfoAt`) + расстояние в км (тайлы ÷ 10).
- **Дозированные хинты** (`hint(id, text, maxShow=3)` в main.js): обучающий toast показывается не более maxShow раз за игру, потом молчит. Счётчик `G.hints` в сейве. Применён к `surf` (нужен водный 15+) и `rod` (нужна удочка).
- **Бой**: покемон-раскладка (враг ↗, свой ↙ со спины — `speciesSprite(...,back)` не рисует глазки). Анимации CSS: `idlebob` (дыхание), `lunge-r/l` (выпад), `shake` — **только классовая специфичность, id-селекторы сломают** (уже наступали). Отменяемые меню имеют кнопку «‹ Назад» (иначе софтлок на таче).
- **Взаимный нокаут = поражение** (фикс 19.07): проверка нокаута врага в `Battle.run` идёт ПЕРЕД проверкой своих и делает `break` — если последний свой боец убивал врага и погибал сам (отдача «Отчаянного удара» при кончившихся ПП либо яд/ожог в `applyEndTurn`), бой возвращал `'win'` с командой на нуле. Дальше: `afterBattle('win')` не лечил и не телепортировал → игрок бродил без братвы, автокач не останавливался, а следующий бой падал с `TypeError` (`firstAlive` = −1 → `party[-1]`), из-за чего встречи молча не запускались. Теперь в ветке последнего врага `result = firstAlive(party) === -1 ? 'lose' : 'win'` (отключка сильнее победы — это же чинит и цикл башни через `result !== 'win'`), плюс страховка в начале `run()`: с полностью павшей братвой бой не стартует и сразу возвращает `'lose'`. Ветка «у тренера ещё есть бойцы» была корректна и раньше (падает в общий блок замены).
- **Разворот маунта** (`drawMountSprite` в main.js): кастомный PNG нарисован смотрящим ВЛЕВО, процедурные спрайты симметричны (`speciesSprite` строит их зеркальными половинами) — поэтому флип нужен ТОЛЬКО кастомным, и проверять надо `spr instanceof Image`, а не `mon.customSprite` (Image из data-URL грузится асинхронно; пока не загрузился, рисуется процедурный фоллбэк). Маунт-братан (`brother`) рисовался вообще без флипа и не разворачивался по ходу — чинено 19.07; водный (`water`) флип имел. Оба теперь идут через общий `drawMountSprite`. Спутник (не маунт) флипается по своему правилу — смотрит на игрока, а не по ходу движения.
- **Кастомные спрайты** (`customSprite` = base64 PNG ≤24×24, смотрит ВЛЕВО): рендер масштабируется к размеру стадии через `monSpriteDims` (12/15/18px); в бою свой флипается вправо, спутник флипается по позиции игрока. `Image` из data-URL грузится асинхронно — `monMiniCanvas` дорисовывает канвас по `load` (иначе видно старый облик до перезахода). Едет в сейв/облако/обмены с другом; в публичный пул НЕ попадает (whitelist воркера).
- **Облачные сейвы**: Telegram CloudStorage, чанки `sv_0..N` + `sv_meta` (≤3500 симв/ключ), дебаунс 20 с, свежий по `ts` побеждает при запуске. Ник: KV-ключ `profile`, дефолт из TG username.
- **Живые соперники**: после боёв команда уходит POST /team (аноним `clientId` из localStorage), 40% тренерских боёв берут чужой снапшот («Тренер Братвы <ник>»), уровни клампятся к местности, статы пересчитывает `tradeMonRevive` (анти-чит). Свой снапшот фильтруется по нику (clientId на каждом устройстве свой, сейв общий — иначе дерёшься сам с собой).
- **Безопасность недоверенного текста**: клички/имена умений из чужих кодов обмена/PvP и импортированных кодов сейва рендерятся через `innerHTML` (много мест). Коды идут МИМО API (checksum+соль `TRADE_SALT` лежат в клиенте — валидный код собирает кто угодно), поэтому клиент — единственный барьер: `safeNick`/`safeMoveName` в `reviveOwnedMon` и `tradeMonRevive` режут `<>&"'` (зеркалят фильтры воркера). Не отключать. Свой ник (`playerNick`) рендерится через `textContent` — не трогать. `tradeDecode`/`importSaveCode` отбрасывают коды сверх лимита длины (300КБ/4МБ) до дорогого декода — защита от вставки-DoS.
- **Побег**: «Бежать» работает и против тренеров/лидеров/башни (та же формула от скорости); тренер не помечается побеждённым — можно переиграть. **Спрашивает подтверждение** (с 19.07 — игроки мискликом вылетали из боя): подменю «Остаться и драться» / «🏃 Да, бежать» + «‹ Назад». Подтверждение сделано меню боя, а НЕ нативным `confirm()`: диалог блокирует события и сломал бы async-поток боя и откат сейвскама. «Да, бежать» намеренно ВТОРЫМ пунктом — повторный тап в ту же точку попадает в «Остаться» (безопасный исход), как и в лавке с кнопкой покупки. Отмена не тратит ход (`continue`).
- **uiConfirm — своё UI вместо нативных confirm()/alert()** (`js/main.js`): промис-диалог поверх `#confirm-panel` (`uiConfirm(title, body, yesLabel, noLabel)` → `Promise<boolean>`), `body` — HTML-строка ИЛИ живой DOM-узел (канвасы спрайтов через `monMiniCanvas` не сериализуются в innerHTML/outerHTML — обязаны остаться тем же элементом). Во время показа `G.state='confirm'` (блокирует `step()`/движение), после ответа восстанавливается прежний `G.state`. Безопасный выбор — ПЕРВОЙ кнопкой (та же логика, что у подтверждения побега). Esc = кнопка «Нет» (`confirm-no`). Заменил нативные `confirm()` для: превью команды тренера/лидера в Nuzlocke (`nzConfirmBattle` в nz.js, строки через `teamPreviewRow` — спрайт+имя вида+уровень+тип, как в Кармане), вход в башню и подъём на этаж (`startTowerRun`). Аналогичный `uiPromptNick(m, fallback)` → `Promise<string>` (`#prompt-panel`, поверх — тот же `teamPreviewRow`, ниже текстовое поле) заменил `prompt()` для обязательной клички нового братишки (`nzForceNick`, дёргается из `nzAfterWild`/`nzOnStarter` — оба стали `async`, вызовы через `await`); пустая строка не проходит валидацию (инлайн-ошибка, диалог не закрывается), Esc — запасное имя от братвы вместо бесконечных ретраев. Остальные `confirm()`/`prompt()` (отмена обмена, отпустить братишку, промокод, ручное переименование в Кармане, эволюция камнем/мегакамнем) НЕ тронуты — не входили в задачу.
- **Нижний клиренс на мобиле** (`--safe-bottom-ui` в index.html = `safe-area-inset-bottom` + 24px): на iPhone жест домашней полоски перехватывает тапы и ВЫШЕ самой полоски, поэтому одного `--safe-bottom` мало — нижние кнопки не нажимались. Переменную используют `#joy-base`, `#touch-actions`, `body.mobile .overlay` (padding-bottom) и отдельно `body.mobile #battle` — у боя СВОЙ padding, который перебивал правило `.overlay` (padding-bottom был 10px, боевые кнопки лежали прямо на полоске; наступали). При добавлении нового интерактива у нижнего края — брать `--safe-bottom-ui`, не `--safe-bottom`.
- **Сейвскам** — премиум-товар `scum` **10⭐** (`scumUnlocked`, KV `scum:<id>`; функции гейтятся `SCUM_ON && scumUnlocked`). Тумблер `SCUM_ON`/`mw-scum` в настройках. Две части:
  - Работает ТОЛЬКО в бою (эмуляторный сейв-стейт, `Battle.battleSave`/`battleReload`, кнопки `#bt-scum` вверху боевого экрана): снапшот ОЗ/ПП/статусов обеих команд + сфер + активных бойцов в память. Откат мутирует существующие объекты (ссылки те же) и реджектит текущий `await` маркером `BATTLE_RELOAD` — цикл боя ловит в `try/catch` и перезапускает ход (`say`/`menu` резолвят/реджектят). Переигрывай любой удар. Вне-боевого сейв/отката НЕТ (убрали намеренно — не нужен).
- PvP-реплей (`pvpShowReplay`) — кнопка «⏭ Скип» (`finish()` вываливает лог и показывает итог сразу).
- **KV-экономия**: разовые покупки при старте — одним запросом `POST /status` (`netCheckStatus` → spr/scoot/scum/vip) вместо запроса на товар. «Живой» соперник (`netFetchRival`) тянется ЛЕНИВО при первом бою (`netTakeRival`), НЕ при старте. Не возвращать предзагрузку/отдельные проверки — жжёт KV-операции (упирались в дневной лимит). **Рейт-лимитер — в памяти изолята** (18.07.2026): раньше `rl:`-счётчики в KV жгли ~700 из 800 записей/день (лимит 1000/день); теперь `Map` в глобале воркера — лимит «мягкий» (изоляты не разделяют память между PoP, распределённый абьюз пробьёт >30 req/min), но KV не трогает; НЕ возвращать счётчики в KV. `lb` в `/team` переписывается только если строка игрока реально изменилась (ts не в счёт) или новичок проходит в заполненный топ. **Горячий поток на D1 (сделано 20.07.2026)**: `t:`/`idx`/`lb`/`pvp:` съезжали в дневной лимит KV (1000 записей/сутки — запись после каждого боя + PvP). Перенесены в D1 (binding DB, лимит 100k записей/сутки): таблицы `teams`/`leaderboard`/`pvp` (`backend/schema.sql`). **Контракт эндпоинтов не менялся** — фронт/`net.js`/`sw.js` не тронуты. Массив `idx` исчез (случайный соперник = `ORDER BY RANDOM()`), лидерборд = `INSERT OR REPLACE` строки вместо read-modify-write всего массива. **TTL у D1 нет** — эмулируется `WHERE ts > порог` в выборках (`msAgo(SNAP_TTL/PVP_TTL)`) + ленивый `DELETE ... WHERE ts < порог` (`Math.random()<0.02`) в `/team` и `/pvp/create`. Все покупки (`unlock:`/`scoot:`/`scum:`/`auto:`/`grind:`/`mon:`/`accs:`/`vip:`/`promo:`) ОСТАЛИСЬ в KV — их не трогали. Данные мигрированы разово через wrangler CLI (KV→`migrate.sql`→D1, 55 команд + 22 строки топа). Рейт-лимитер по-прежнему в памяти изолята (KV/D1 не трогает).
- **Маунты** (`activeMount()` в main.js): под игроком рисуется маунт и множится скорость. `water` — плывём на водном братишке 15+ (пассивно, по факту нахождения на воде + визуал качки); `scooter` — премиум-самокат на суше ×1.8 (`scootUnlocked` + `G.scootOn`); `brother` — лидер-братан (stage 2) на суше ×1.5. **Наземный маунт — только при нажатом ускорении** (`keys.has('Shift')`/кнопка бега): пешком идёшь сам, нажал бег — едешь на доступном маунте. Кнопка бега `#t-run` меняет иконку в `updateHUD`: 🛴 самокат / 🐎 братан / 🏃 просто бег (`landMountKind()`). Приоритет вода → самокат → братан; при water/brother спутник не бежит рядом (он и есть маунт).
- **Автобой** (премиум `auto` 50⭐, флаг `autoUnlocked`): кнопки `#bt-autobox` (▶/⏸ Авто и ×1→×2→×3) слева вверху боя, видны только купившим (`updateAutoBtns`). Состояние `Battle._auto`/`_speed` липкое в localStorage `mw-auto-on`/`mw-auto-speed` (НЕ в сейве). Логика: в цикле run() при `_auto` действие всегда «Атака» (v1: без зелий/сфер/побега), умение — `autoPickMove` (приоритет `eff*10000 + power×STAB×acc` — эффективность типа доминирует: ×2 > ×1 > ×½), замена после нокаута — `autoPickSwitch` (лучшая эффективность умений против текущего врага, tie → уровень). `say()` сам листает через `650/_speed` мс (клик тоже работает), пауза удара 150мс делится на скорость. Вкл посреди меню действий — `setAuto` дожимает «Атака» через `_menuKind === 'actions'`; прочие открытые меню игрок довыбирает вручную. Сейвскам-откат совместим (реджекты say ловятся как раньше).
- **Автокач** (премиум `grind` 100⭐, бандл поверх автобоя; флаг `grindUnlocked`, код в main.js перед `step()`): `setGrind`/`grindVector`/`grindZoneAt`, `GRIND_ON` — runtime-флаг (НЕ в сейве). Включение: тач-кнопка 🤖 `#t-grind` (видна купившим в дикой зоне, `updateHUD`) или клавиша G; дозированный `hint('grind')` при входе в зону. Зоны `GRIND_ZONES` — группы тайлов: вода / трава+TALL+FLOWER / снег / пустыня; старт фиксирует группу, бег только по ней, города (`cityInfoAt`) исключены. Вектор: случайный курс на 0.8–2с с проверкой `ahead` (тайл в 0.8 и 1.4 впереди в зоне), перевыбор при застревании (позиция не сместилась за 0.5с). Бои триггерятся штатно (`onTileEnter`-энкаунтеры; бамп в тренера — тренерский бой), при старте автокача принудительно `Battle.setAuto(true)`. Стоп: любой ручной ввод (клавиши/джойстик), выход из зоны (в т.ч. телепорт), lose в `afterBattle` (респ у фонтана). Ловли в автокаче нет by design (автобой всегда жмёт «Атака»).

Гайд (H) в Nuzlocke открывается с секцией «☠️ Правила Nuzlocke» первой, раздел «Питомник» отсутствует. Обычный режим не гейтится ничем из этого — обменники продолжают спавниться, Предметы в бою активны, HUD без «макс.лвл»/«📍». **Stars-кнопки скрыты вне Telegram в обоих режимах** (`IS_TMA` гейтит самокат/сейвскам/автобой/автокач — disabled «🔒 Только в Telegram», автокач вдобавок требует купленный автобой). **Главы летописи в чат — хронология и порядок (20.07)**: по фидбоду владельца (главы «Потеря» тащили в тексте 2 случайных соседних события, а при массовой гибели братвы за один бой открытки прилетали в чат вразнобой, не по порядку — ломало составление фанфика). Причины были две: (1) `nzChapter` брал `G.nz.log.slice(-3)` — фиксированные «последние 3» вместо реально случившегося с прошлой главы; (2) `netNzChapter` (net.js) — fire-and-forget `fetch` без ожидания, параллельные запросы к воркеру могли доехать до Telegram в другом порядке. Фикс: каждая запись лога получает `e.rep=1` после того как попала хоть в одну главу, пересказ = `G.nz.log.filter(e => !e.rep)` (переживает перезапуск мини-аппа — флаг в самих данных, не в переменной сессии) — при тротлинге (не чаще раза в 30с) пропущенные события не теряются, а целиком уходят следующей главой (напр. блэкаут-открытка утащит все смерти между собой и предыдущей отправленной главой). `netNzChapter` теперь возвращает промис fetch, `nzChapter` сериализует отправку через `_nzChapterQueue` (цепочка `.then`) — гарантирует прибытие в чат в порядке вызова, а не в порядке ответа сети. sw.js v54.
