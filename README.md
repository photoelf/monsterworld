# 🐾 Мир Монстриков

Браузерная игра в жанре «лови и сражайся» с **бесконечным процедурным миром** и полностью процедурными монстриками. Без движков и зависимостей — чистый HTML/JS/Canvas.

**Играть:** https://photoelf.github.io/monsterworld/

## Что внутри

- **Бесконечный мир из сида**: поля, леса, парки, города, снега и пустыни; время суток и погода влияют на фауну и бой.
- **Процедурные монстрики**: имя, спрайт, тип, статы, умения и цепочка эволюций (1–3 стадии) — всё из одного числа. ✨ Сияющие — 1 из 64.
- **Бои**: 8 типов с таблицей эффективности, статусные эффекты (яд/ожог/паралич/сон/заморозка), ПП умений, предметы.
- **Прогрессия**: тренеры, арены со значками, башня испытаний, легендарные святилища, гнёзда, доска заданий, достижения.
- **Хозяйство**: лавки, амулеты, свитки умений, камень эволюции, питомник с яйцами, NPC-обменники, рыбалка, сёрфинг.
- **Перенос**: сейв-код (base64) — продолжай на любой машине.
- **📦 Монстрохранилище**: безлимитный ящик — пойманные при полной команде не теряются; из хранилища можно брать в команду, для яиц и обменов.
- **🤝 Обмен с другом**: асинхронный трёхшаговый трейд по кодам с эскроу и защитой от подмены.
- **⚔️ PvP по кодам**: обменяйтесь командами — обе стороны увидят один и тот же детерминированный бой; результат подделать нельзя, каждый считает его сам.
- **📱 PWA**: ставится на телефон, работает офлайн; на тач-устройствах — экранная крестовина и кнопки.

## Управление

| Клавиша | Действие |
|---|---|
| WASD / стрелки | Ходить |
| Shift | Бег |
| Tab | Команда |
| B | Монстрохранилище |
| M | Карта (+телепорт к фонтанам, код сейва) |
| P | Монстропедия |
| O | Достижения |
| T | Обмен с другом / PvP |
| F | Рыбалка (нужна удочка) |
| 1–4 / Enter / Esc | Меню боя |

## Запуск локально

Просто открой `index.html` в браузере. Сервер не нужен.

## Как устроен обмен с другом

1. Игрок А создаёт предложение — его монстрик уходит в эскроу, генерируется код-оферта.
2. Игрок Б вставляет код, видит монстрика, выбирает своего взамен (тоже эскроу) — получает код-ответ.
3. А вставляет ответ, подтверждает — получает монстрика Б и финальный код.
4. Б вставляет финальный код — игра сверяет хэш обещанного монстрика и выдаёт его.

Коды защищены контрольной суммой, сделки одноразовые, статы пришедшего монстрика пересчитываются из его вида — «нарисовать» имбу в коде нельзя.
