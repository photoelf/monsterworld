# Nuzlocke-режим («☠️ Nuzlocke») — дизайн

Дата: 2026-07-20. Статус: утверждён владельцем (детали ниже), реализация по варианту A (флаг режима + точечные гейты).

## Цель

Хардкорный игровой режим по канону Nuzlocke Challenge: пермасмерть, правило первой встречи,
обязательные клички, запрет предметов в бою, Set-режим, левел-кап — плюс автоматическая
«Летопись» рана (готовый фанфик о похождениях Братвы) с отправкой глав в чат Telegram-бота.
Премиум-бонусы за звёзды в режиме не работают.

## Слоты сейва

- **Два независимых слота**: основной и Nuzlocke. Активный слот — `localStorage('mw-slot')`
  (`'main'` | `'nz'`).
- Локальный сейв: текущий ключ — для main; тот же ключ с суффиксом `-nz` — для Nuzlocke.
- Облако (Telegram CloudStorage): main — чанки `sv_0..N` + `sv_meta` (как сейчас);
  Nuzlocke — `nz_0..N` + `nz_meta`. Логика чанкования/дебаунса/`ts`-гонки общая,
  параметризуется префиксом.
- В самом сейве — `G.nuzlocke = true` и блок `G.nz` (см. ниже). Хелпер `NZ()` в main.js.
- Титул: если есть Nuzlocke-сейв — кнопка «☠️ Продолжить Nuzlocke» рядом с «Продолжить».
  «Новая игра» → чекбокс «☠️ Режим Nuzlocke»: при включении раскрывается краткий свод правил
  и предупреждение «купленные за звёзды бонусы здесь не работают». Старт NZ-рана не трогает
  основной сейв; старт обычной новой игры не трогает NZ-слот.

## Структура `G.nz`

```js
G.nz = {
  cap: 15,             // текущий левел-кап
  zones: {},           // zoneKey -> 'used' (отсутствие ключа = встреча доступна)
  lineages: [],        // сиды базовых видов пойманных эво-линий (для Dupes Clause)
  graveyard: [],       // павшие: {mon, caughtLvl, diedLvl, battles, killer, place, ts}
  log: [],             // события летописи: {t, k, ...данные} — БЕЗ текста
  stats: {}            // счётчики рана: поимки, смерти, лидеры, бои
}
```

## Зоны первой встречи («области»)

- Зона = Вороной-ячейка города (существующая сетка 96×96 в world.js). Граница области —
  «половина пути между городами», как и хотел владелец.
- `zoneKey` = координата ячейки (`cx,cy`). **Мёртвые ячейки** (центр на воде, города нет)
  прикрепляются к ближайшему живому городу — детерминированная функция `nzZoneAt(x,y)`,
  у бесхозных территорий хозяин всегда один и тот же.
- Имя области = «обл. <Имя города>».

### Правило первой встречи

- Первый дикий бой в области (случайная встреча, гнездо, святилище, рыбалка) = «встреча зоны»:
  только в нём доступна кнопка «Поймать». Любой исход (поймал / убил / сбежал / проиграл /
  дикий добил) → зона помечается `used`.
- **Dupes Clause**: если выпавший вид принадлежит эво-линии, уже пойманной в этом ране
  (`nz.lineages`), бой встречей НЕ считается: «Поймать» недоступна (подпись «такой уже есть
  в семье»), зона остаётся свежей — ждём первый новый вид.
- **Shiny Clause**: шайни можно ловить ВСЕГДА — и в использованной зоне, и при дупе.
- Во всех прочих диких боях «Поймать» скрыта/заблокирована с причиной.
- Репеллент разрешён (предмет вне боя, встречи он глушит честно).
- Башня — не дикая зона, ловли там и так нет.

### Индикация

- HUD в дикой зоне: имя области + «🎯 встреча доступна» / «✔ использована».
- **Карта**: рисуем границы областей, в каждой — имя города и значок 🎯/✔.
  Реализация: на разрешении карты для каждого пикселя/группы пикселей считается ближайший
  живой город; граница там, где сосед отличается. Дёшево, кэшируется на рендер карты.

## Пермасмерть и Кладбище

- После боя каждый братишка из братвы с 0 ОЗ **умирает**: переносится из party в
  `nz.graveyard` с данными {уровень поимки, уровень смерти, число боёв (`m.nzBattles`,
  инкремент за каждый бой с участием), убийца (вид/тренер), место (область/этаж башни), дата}.
- Взаимный нокаут: правило «отключка сильнее победы» уже есть — павшие умирают и при 'win'.
- Фонтан лечит только живых. Воскрешений нет нигде.
- **Вкладка «⚰️ Кладбище» в Кармане** (видна только в NZ): грейскейл-спрайты
  (canvas filter `grayscale(1)`), эпитафия: «Колюня. Прожил с 5 по 23 ур., 47 боёв.
  Пал от Мега-Хмыря, башня, 12-й этаж».

## Прочие правила

- **Клички обязательны**: после поимки — диалог клички без возможности пропустить.
- **Скорость тренера ×0.5** — пешком, бегом, на любом маунте (реже влетаешь в ненужные бои).
- **Спавн диких ×0.3** — множитель на шанс случайной встречи в `onTileEnter`.
- **Предметы в бою запрещены**: боевое меню «Предметы» скрыто в NZ. Кнопка «Поймать»
  (сферы) остаётся — по канону ловля разрешена. Вне боя зелья/эфиры/тоники работают.
- **Set-режим**: бесплатных замен при нокауте врага быть не должно. В текущем бою
  предложения смены при смерти вражеского бойца нет (проверить при реализации) — если
  найдётся, в NZ отключить.
- **Поражение** (есть живые в кармане): как обычно −50% денег + телепорт к фонтану;
  павшие в том бою — на кладбище; братву доукомплектовываешь из кармана.

## Левел-кап

- Старт: **15**.
- После победы над лидером арены: `cap = уровень туза слабейшего НЕпобеждённого лидера
  среди открытых (посещённых) городов`. Туз вычисляется детерминированно из уровня города
  (`levelAt` центра, как формируется команда лидера).
- Если непобеждённых открытых лидеров нет — `cap = туз только что побеждённого + 10`
  (стимул идти открывать новые города; при открытии нового города кап пересчитывается).
- Кап только растёт (пересчёт берёт `max(текущий, новый)`).
- Опыт сверх капа **сгорает**: `grantExp` в NZ клампит уровень капом. Эволюция возможна,
  если `evolveLevel ≤ cap`.
- HUD: «⛔ N».

## Blackout (Game Over)

- Условие: после боя вся братва И весь карман мертвы.
- Экран итогов: полная Летопись рана + статы (дней/боёв/поимок/смертей/лидеров),
  кнопки «Поделиться летописью» и «Начать новый ран».
- «Начать новый ран» стирает NZ-слот; текст летописи погибшего рана сохраняется локально
  (`mw-nz-lastrun`) до следующего Game Over.

## Что отключено / оставлено в NZ

Отключено (UI скрыт, флаги игнорируются, VIP не спасает):
- сейвскам, автобой, автокач, самокат, генератор заказных братишек;
- обмены кодами, PvP (обе ветки), питомник;
- достижения и лидерборд (UI); снапшот своей команды НЕ шлётся в `/team`
  (гейт в `netSendTeam`), в `lb` NZ-игрок не попадает.

Оставлено:
- **живые соперники** (40% тренерских — чужие снапшоты; read-only, `netTakeRival` работает);
- гардероб/косметика/перекраски;
- **«Сменить спрайт» — бесплатно всем в NZ** (фича premium `spr` открыта внутри режима
  для привязанности к братишкам; на основной слот не влияет, KV не пишется);
- башня, мегаэволюция, лавка (витрины премиума в «Особом» скрыты), доска заданий, рыбалка.

## Ростер (панель «Братва»)

В NZ кнопки «Достижения» и «Обмен/PvP» заменяются кнопкой «📜 Летопись».

## Летопись (фанфик)

- В сейве — только компактные события `nz.log` (`{t, k, ...}`), текст генерится при рендере.
- События: старт рана, встреча зоны (исход), кличка, эволюция, победа над лидером /
  супертренером, смерть, веха (5/10/… поимок, этажи башни), blackout.
- Текст — процедурный, детерминированный по событию (сид), тёплый уличный тон в духе
  `speciesFlavor`: «В обл. Краснореченск братва встретила ежа. Назвали Колюня.
  Колюня влился в семью». Смерти — драматично, с именем убийцы.
- Панель «📜 Летопись»: лента глав (свежие снизу), кнопка «Поделиться» (share/копировать
  весь текст).

### Главы в чат бота (только TMA)

- Майлстоны: смерть, победа над лидером, blackout (первая поимка рана — тоже).
- Клиент: скрин боевой сцены `canvas.toDataURL('image/jpeg', ~0.7)` в момент исхода
  (≈50–100 КБ) + текст главы → `POST /nzlog` {initData, photo, text}.
- Воркер: верификация initData (HMAC), рейт-лимит в памяти изолята, `sendPhoto`
  (multipart) в чат `verifyInitData().id`. **KV не трогаем вообще.** Best-effort:
  ошибка отправки не ломает игру.
- Тумблер в настройках «📖 Слать главы в чат» (`mw-nz-chapters`, дефолт вкл в TMA;
  вне TMA фича недоступна — скрины живут только в чате, в сейв не пишутся).
- Порядок деплоя: **воркер раньше фронта**.

## Тестирование

- `node --check js/*.js`; бамп `CACHE` в sw.js.
- Playwright: (1) старт NZ при живом основном сейве → переключение туда-обратно без потерь
  (оба слота целы, облачные ключи не пересекаются); (2) встреча зоны: ловля доступна только
  в первом бою, dupes пропускает, шайни ловится всегда; (3) смерть → кладбище с эпитафией,
  грейскейл; (4) blackout → экран итогов с летописью; (5) кап режет опыт, победа над
  лидером поднимает кап; (6) `/nzlog` с мок-TMA (initData подписывается питоном, как в
  существующих тестах).
